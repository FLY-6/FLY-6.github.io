# 博客内容修改与技术实现说明

## 1. 博客内容如何修改

### 1.1 文章文件位置

所有文章 Markdown 文件存放在：

- `content/blog/`

例如：

- `content/blog/git-workflow-best-practices.md`
- `content/blog/nextjs-14-guide.md`

### 1.2 文章文件格式

每篇文章都必须以 YAML frontmatter 开头，包含如下字段：

```md
---
title: "文章标题"
date: "2026-05-01"
excerpt: "文章摘要，用于列表预览。"
tags: ["标签1", "标签2"]
category: "分类"
author: "作者"
readingTime: 5
---

## 正文标题

文章内容...
```

字段说明：

- `title`：文章标题。
- `date`：文章日期，建议使用 `YYYY-MM-DD` 格式。
- `excerpt`：文章摘要，用于首页和列表页展示。
- `tags`：标签列表，用于统计和展示。
- `category`：文章分类，用于筛选。
- `author`：作者名。
- `readingTime`：阅读时长，页面会直接显示该值。

### 1.3 添加新文章

1. 在 `content/blog/` 目录下创建一个新的 `.md` 文件。
2. 添加 YAML frontmatter 和 Markdown 正文内容。
3. 保存后，Next.js 的静态构建会自动读取该文章。

### 1.4 修改文章

直接编辑对应 `content/blog/*.md` 文件即可。

> 如果你使用 GitHub Pages 自动部署，修改后推送到 `main` 分支即可触发构建和更新页面。

## 2. 技术实现概览

### 2.1 数据读取与解析

文章数据读取逻辑在：

- `src/lib/posts.ts`

该文件实现了：

- `getAllPostSlugs()`：读取 `content/blog` 目录，返回所有文章 slug。
- `getAllPosts()`：读取所有文章并按照日期降序排序。
- `getPostBySlug(slug)`：根据 slug 读取单篇文章内容和 frontmatter。
- `markdownToHtml(markdown)`：使用 `remark` + `remark-html` 将 Markdown 转为 HTML。
- `getAllTags()`：统计标签数量。
- `getPostsByCategory()`：按分类分组文章。
- `searchPosts(query)`：根据关键字搜索文章标题、摘要、标签、分类和正文。

### 2.2 页面路由结构

该项目使用 Next.js App Router，页面结构如下：

- `/`：首页，位于 `src/app/page.tsx`
- `/about`：关于页，位于 `src/app/about/page.tsx`
- `/blog`：博客列表页，位于 `src/app/blog/page.tsx`
- `/blog/[slug]`：文章详情页，位于 `src/app/blog/[slug]/page.tsx`

### 2.3 首页实现

首页 `src/app/page.tsx` 读取：

- `getRecentPosts(6)`：首页最新文章。
- `getAllTags()`：标签云。
- `getAllPosts()`：文章总数。

然后将数据渲染到首页组件中。

### 2.4 博客列表页实现

博客列表页现在拆成两部分：

- `src/app/blog/page.tsx`：服务器组件，负责读取文章数据。
- `src/app/blog/BlogListClient.tsx`：客户端组件，负责搜索、分类过滤和交互。

这样可以避免在客户端组件中直接导入 `fs` 模块导致构建失败。

### 2.5 文章详情页实现

文章详情页 `src/app/blog/[slug]/page.tsx`：

- 使用 `generateStaticParams()` 生成静态路径。
- 使用 `getPostBySlug(slug)` 读取文章数据。
- 使用 `markdownToHtml()` 将 Markdown 转换为 HTML。
- 通过 `dangerouslySetInnerHTML` 渲染文章内容。

## 3. GitHub Pages 部署相关

### 3.1 Next.js 静态导出配置

项目使用 `next.config.js` 配置：

- `output: "export"`
- `trailingSlash: true`
- `basePath` 和 `assetPrefix` 支持 GitHub Pages 子路径

### 3.2 部署工作流

仓库中已添加 GitHub Actions 配置：

- `.github/workflows/deploy.yml`

该工作流会：

1. 安装依赖
2. 执行 `npm run build`
3. 上传 `out/` 静态文件
4. 使用 `actions/deploy-pages@v5` 部署到 GitHub Pages

### 3.3 项目页与用户页区别

- 用户页仓库名称形如 `username.github.io`，访问路径为：
  - `https://username.github.io`
- 项目页仓库名称为 `repo`，访问路径为：
  - `https://username.github.io/repo`

工作流已自动检测仓库名，并在项目页情况下设置 `basePath`，因此一般不需要手动配置额外 secret。

## 4. 修改博客功能时需要注意的点

### 4.1 禁止在客户端组件中直接使用 `fs`

由于 Next.js 客户端组件无法访问服务器端 API，不能在 `use client` 文件里直接导入 `fs` 或 `path`。

如果需要读取 Markdown 文件，只能在服务器组件中完成，然后将结果传给客户端组件。

### 4.2 文章列表搜索与过滤

搜索功能在 `src/app/blog/BlogListClient.tsx` 内实现。

如果想增强搜索：

- 可以在 `src/lib/posts.ts` 中调整 `searchPosts()` 条件
- 或在客户端组件内增加更复杂的过滤逻辑

### 4.3 添加新页面或功能

如果想新增页面或分类页：

- 在 `src/app/` 中创建新目录与 `page.tsx`
- 使用 `src/lib/posts.ts` 提供的数据接口读取文章
- 对 SEO、元数据可在页面组件中按需补充 `metadata`

## 5. 进一步扩展建议

### 5.1 增加文章封面图

可以在 frontmatter 中添加 `coverImage` 字段，并在文章卡片、详情页中展示。

### 5.2 增加分页功能

当前 `/blog` 页面会展示所有文章。如果文章数量较多，可考虑：

- 在 `BlogListClient.tsx` 中实现前端分页
- 或在服务器端生成分页页面

### 5.3 增加更多分类筛选

`category` 字段已经支持分类过滤。你可以：

- 在首页增加分类入口
- 在列表页增加侧边栏分类树
